# PGB1 项目 Onboarding

你正在进入 PGB1 项目的开发会话。请按以下顺序执行：

## 铁律：文档优先，禁止盲搜

**定位代码时必须先查文档，严禁跳过文档直接搜索代码。**

- `INDEX.md` — 全局索引，找到设计文档和关键决策的位置
- `CODE_INDEX.md` — 全量代码文件职责索引（文件名/行数/职责/命令列表/行号），直接定位目标文件
- 正确流程：INDEX.md → CODE_INDEX.md → Read 目标文件
- 错误流程：~~Grep/Glob/Explore agent 盲搜~~

文档已经标注了每个文件的职责、行数、关键接口，盲搜是浪费时间。

## 步骤 1：读取全局索引

读取 `INDEX.md`，了解：
- 项目概要和当前状态
- 所有设计文档的位置和行号索引
- 关键决策速查表
- 待明确事项

## 步骤 2：读取代码索引

读取 `CODE_INDEX.md`，建立代码全貌认知：
- 所有源文件的职责和行数
- Rust 后端 61 个命令的参数/返回值/职责
- 关键架构模式（数据流、状态管理、样式 SSOT 等）

## 步骤 3：确认开发规范

检查 `开发规范.md` 是否存在，如果存在则读取其中的代码规范和目录结构要求。

## 步骤 4：检查项目代码状态

如果项目已有代码（`src/` 目录存在），快速扫描当前代码结构，了解已实现的部分。

## 步骤 5：输出就绪摘要

向产品总监简要汇报：
- 当前项目阶段
- 已实现 vs 待实现的功能（如果有代码的话）
- 准备好接受什么类型的任务

**语言**：所有输出使用简体中文。
**原则**：简洁直接，不要长篇大论。

## 🌳 任务架构

### 树状结构

任务采用**树状结构**组织：
```
项目：217_RedDevil
├─ Free Spin特效 [父任务]
│   ├─ 普通版本 [子任务]
│   ├─ 金色版本 [子任务]
│   └─ 特殊版本 [子任务]
├─ Win Highlight [父任务]
│   ├─ 小赢 [子任务]
│   ├─ 中赢 [子任务]
│   └─ 大赢 [子任务]
└─ Ambient [叶子任务，无子任务]
```

### 任务类型

**父任务**：
- 包含多个子任务
- 显示子任务完成进度（如"状态 1/3"）
- 所有子任务共享同一个文件夹

**子任务**：
- 隶属于父任务
- 有独立的名称
- 制作时间不同步
- 最终素材导入同一文件夹

**叶子任务**：
- 无子任务的独立任务
- 直接管理素材

### 核心特点

1. **共享文件夹**：父任务的所有子任务最终导入同一文件夹
2. **独立进度**：子任务制作时间不同步，可分批完成
3. **不关联素材**：素材不与具体子任务关联，只统计总体上传状态

**视觉设计** → 参考 `卡片设计.md` - 任务卡片

## 📋 任务模板系统

### 模板定义

**全局任务清单**：
- 存储位置：`~/.pgb1/global_tasks.json`
- 包含所有可用的任务和子任务模板
- 用户可添加/删除/编辑任务和子任务

**项目任务配置**：
- 存储位置：`.pgb1_project.json`（项目根目录）
- 记录当前项目启用的任务
- 记录归档的任务（`.archived_tasks/`）

### 默认模板

**已确定的默认任务清单**：

1. **ambient** - 环境特效（无子任务）
2. **free spin** - 免费旋转（4个子任务）
   - slow drop
   - fast spin
   - scatter
   - freespin retrigger
3. **infoboard** - 信息板（2个子任务）
   - infoboard
   - one more scatter
4. **main ui** - 主界面UI（无子任务）
5. **mouse hover** - 鼠标悬停（无子任务）
6. **prototype** - 原型素材（无子任务）
7. **spin button** - 旋转按钮（无子任务）
8. **win highlight** - 赢得高亮（2个子任务）
   - small win
   - wild

**注意**：
- 这是默认模板，用户可以通过模板管理功能添加/删除/修改
- 任务名称使用小写英文（符合实际项目文件夹命名）

### 模板管理功能

**操作**：
- ✅ 添加新任务/子任务
- ✅ 删除任务/子任务（带保护机制）
- ✅ 重命名任务/子任务（全局同步）
- ✅ 编辑任务的子任务结构

**管理界面** → 详见 [任务管理窗口](#任务管理窗口)

**文件系统规则** → 参考 `文件命名与组织规则.md`

## 🎯 子任务管理

### 创建位置

**在项目页的任务列表中创建**：
- 用户打开项目页
- 点击 `[任务列表]` 按钮
- 在弹出的任务管理窗口中选择任务
- 支持启用父任务（自动包含所有子任务）

**界面布局** → 参考 `界面设计.md` - 第二级：项目页

### 显示方式

**已确定**：类似代码的tab缩进，直接全部展示（不需要折叠）

**显示示例**：
```
win highlight        on/off
  - scatter          on/off
  - wild feature     on/off

free spin            on/off
  - slow drop        on/off
  - fast spin        on/off
  - scatter          on/off
  - freespin retrigger on/off

ambient              on/off
```

**设计要点**：
- 父任务和子任务都显示在项目页的任务列表中
- 子任务通过缩进（如：前置"-"或缩进空格）与父任务区分
- 每个任务/子任务都有 on/off 开关（启用/禁用）
- 不需要展开/折叠交互，始终全部可见

### 点击行为

**已明确**：
- **点击任务（无论父任务还是子任务）** → 进入该任务的任务页
  - **重要**：任务页面不分父任务和子任务
  - 一个任务只有一个任务页面
  - 所有子任务的素材都在同一个任务页面里混合显示
  - 子任务只是进度追踪工具，不是独立页面
- **on/off 开关** → 切换任务的启用/禁用状态，不触发页面跳转

### 子任务进度管理

#### 任务页的子任务按钮

**位置**：任务页的快捷功能区

**显示内容**：
- 按钮文字：`[子任务 1/3]` 或 `[子任务进度 1/3]`
- 显示当前已完成的子任务数/总子任务数

**点击行为**：
- 弹出弹窗，显示所有子任务列表
- 用户可以手动勾选/取消勾选子任务的完成状态
- 修改后自动更新按钮显示

**自动判定逻辑**：
- 页面刷新时，检测页面内的素材上传情况
- 如果检测到所有素材都上传，弹窗询问用户："哪个子任务完成了？"
- 用户选择后，自动更新子任务进度

**界面设计** → 详见 `界面设计.md` - 任务页快捷功能区

## 🔄 进度判定逻辑

### 检测触发时机

**页面刷新时检测**：
- 用户刷新任务页
- 系统自动扫描素材上传状态

### 判定规则

#### 1. 无子任务（叶子任务）

```
触发：页面刷新
检测：所有素材 + 所有预览视频是否都上传到nextcloud
结果：
  - 是 → 自动标记任务完成（状态变绿）
  - 否 → 保持当前状态
注：无预览视频时，仅以素材判定
```

#### 2. 有子任务（父任务）

```
触发：页面刷新
检测：所有素材 + 所有预览视频是否都上传到nextcloud

情况A：所有素材 + 预览视频已上传
  ↓
弹窗询问："检测到所有素材已上传，哪个子任务完成了？"
  ↓
用户选择："普通版本"
  ↓
更新状态："状态 1/3" → "状态 2/3"
背景色：黄色（进行中）
  ↓
所有子任务完成后（3/3）
  ↓
背景色变绿（已完成）

情况B：条件从满足变为不满足，且已有≥2个子任务被勾选
  ↓
弹窗询问："检测到素材状态变化，是否更新状态？"
  ↓
用户选择：
  - 取消某个子任务的完成状态（如：2/3 → 1/3）
  - 保持不变（忽略警告）
```

### 素材上传状态检测

**检测逻辑**：
- 素材：扫描 `00_original/`，对比 `nextcloud/{任务名}/` 目录
- 预览视频：扫描 `03_preview/`，对比 `nextcloud/preview/`（breakdown 文件对比 `nextcloud/preview/breakdown/`）
- 两者均满足才判定完成；无预览视频时仅以素材判定
- "outdated"（旧版本在 nc，新版本未传）视为未完成

**上传状态同步机制** → 参考 `文件命名与组织规则.md` - nextcloud同步规则

### 状态背景色规则

**有子任务（父任务）**：

| 状态 | 条件 | 背景色 |
|------|------|--------|
| 未开始 0/N | 0 个子任务完成 | 灰色 |
| 进行中 X/N | 0 < X < N | 黄色 |
| 已完成 | X = N | 绿色 |

**无子任务（叶子任务）**：

| 状态 | 条件 | 背景色 |
|------|------|--------|
| 未开始 | 素材数 = 0 | 灰色 |
| 制作中 | 有素材且未全部上传，或预览视频未全部上传 | 蓝色 |
| 已完成 | 所有素材 + 预览视频已上传（无视频时仅需素材完成） | 绿色 |

**视觉实现** → 参考 `卡片设计.md` - 任务卡片

## 🪟 任务管理窗口

### 触发方式

**入口**：项目页的 `[任务列表]` 按钮

### 窗口类型

**已确定**：模态弹窗

### 窗口内容

**Tab 1：当前项目任务启用管理**
- 显示全局任务清单
- 勾选框：启用/禁用任务
- 显示已启用任务的状态

**Tab 2：全局任务清单模板管理**
- 显示所有任务和子任务
- 操作：添加、删除、重命名、编辑子任务
- 删除保护机制（如果任务已被使用，警告用户）

**Tab 3：时光机（恢复归档任务）**
- 显示归档任务的历史版本列表
- 显示归档时间
- 操作：选择版本并恢复
- 自动清理超过60天的归档

**详细规则** → 参考 `文件命名与组织规则.md` - 任务管理系统章节

### 操作行为

**重命名任务**：
- 修改全局任务清单中的任务名称
- 自动重命名所有已有项目中该任务的文件夹
- 同步更新所有项目的配置文件

**删除任务**：
- 如果任务已在某个项目中使用，弹窗警告
- 用户确认后，从全局清单删除
- 已启用该任务的项目不受影响（已创建的文件夹保留）

**归档任务**：
- 在项目中关闭任务时，移动到 `.archived_tasks/{任务名}_{时间戳}/`
- 支持多次归档（通过时间戳区分）
- 60天后自动清理

## 🎨 各卡片详细设计

### 1. 项目卡片

**布局结构**:
```
┌─────────────────────────┐
│ ┌─────┐  项目名         │
│ │ICON │  ─────────────  │
│ │     │  截止日期       │
│ └─────┘  ─────────────  │
│ ▓▓▓▓▓▓░░░░░░░░ 进度条  │
└─────────────────────────┘
```

**信息字段**:
- **ICON**（左侧圆角矩形区域，80×80px）
  - 来源：`01_Preproduction/` 下文件名含"appicon"（大小写不敏感）的文件，PNG 优先，其次 PSD/PSB
  - 备选：找不到时显示默认文件夹 SVG 图标（灰色背景）
  - 渲染：PNG 用 `convertFileSrc` 直显；PSD/PSB 走 `usePsdThumbnail` 异步提取 128px 缩略图
  - 样式：`object-fit: cover` 放大 110% 填满容器，`overflow: hidden` 裁切，有图片时叠加 3px 白色内描边
- **项目名**（右上方）
  - 格式：`{项目编号}_{项目代号}`
  - 示例：`217_RedDevil`
- **截止日期**（右下方）
  - 格式：YYYY-MM-DD（如：2026-03-15）
  - 数据来源：用户创建项目时手动输入，后续可在项目设置中修改
- **进度条**（底部）
  - 计算方式：分母 = 无子任务父任务数 + 所有子任务数；分子 = 已完成数（无子任务父任务看 nextcloud 全上传，有子任务父任务看 completed_subtasks）
  - 样式：4px 高度，蓝色填充（`--color-primary-500`），100%完成时切换为绿色（`--color-success`，与已完成标签一致），颜色切换带平滑过渡动画；右侧显示「X / Y 个任务」文字

**交互行为**:
- 悬停：放大 + 高亮（参数待细调）
- 点击：进入该项目的项目页
- 拖拽：支持拖拽到外部，不允许应用内位置调整

**布局结构**:
```
┌─────────────────────────┐
│ 任务名                  │
│ ───────────────────────│
│ ┌───────────┐    50Kb  │
│ │状态  1/3  │          │
│ └───────────┘          │
└─────────────────────────┘
```

**信息字段**:
- **任务名**（顶部）
  - 显示完整任务名称
  - 如果过长，使用省略号（...）
- **状态标签**（左下方，椭圆形）
  - 规则因任务类型而异（详见下方说明）
- **文件大小**（右下方）
  - 显示当前任务文件夹的总文件大小
  - 格式：自动单位转换（Kb/Mb/Gb）

**状态标签说明**：

**有子任务（父任务）**：分母 = 子任务数，3态

| 状态 | 条件 | 颜色 |
|------|------|------|
| 未开始 0/N | 0 个子任务完成 | 灰色 |
| 进行中 X/N | 0 < X < N | 黄色 |
| 已完成 | X = N | 绿色 |

**无子任务（叶子任务，如 Prototype）**：不显示数字，3态

| 状态 | 条件 | 颜色 |
|------|------|------|
| 未开始 | 素材数 = 0 | 灰色 |
| 制作中 | 有素材且未全部上传 | 蓝色 |
| 已完成 | 所有素材已上传 | 绿色 |

**子任务系统详细说明** → 参考 `任务系统.md`

**交互行为**:
- 悬停：放大 + 高亮
- 点击：进入该任务的任务页
- 拖拽：支持拖拽到外部

## 🖼️ 预览图生成规则

### ⚠️ 数据源规则

**任务页素材卡片的预览图必须始终使用 `00_original/` 中的素材**

- **原因**：确保预览图展示的是原始质量，而非缩放或转换后的版本
- **适用范围**：任务页的素材卡片
- **技术实现**：卡片渲染时，优先从 `00_original/` 读取文件，而不是从 `01_scale/` 或 `02_done/` 读取

---

### 预览方式对照表

| 文件类型 | 预览方式 | 数据源 | 技术实现 |
|---------|---------|--------|---------|
| **图片类** | 显示图片内容 | `00_original/` | 直接加载图片文件 |
| **序列帧** | 显示动画预览 | `00_original/` 序列帧文件夹 | Canvas实时渲染（详见下文） |
| **精灵图** | 沿用对应序列帧的缩略图动画 | `00_original/` 对应序列帧 | 识别对应的序列帧源文件，使用其动画 |
| **视频** | 显示视频内容 | `00_original/` | 提取视频缩略图或内嵌播放器 |
| **文档等其他** | 通用图标 | N/A | 根据文件扩展名显示对应的图标 |

### 序列帧预览详细规范

#### 数据来源
- **源文件**: `00_original/` 目录中的序列帧PNG文件
- **帧率优先级**:
  1. TexturePacker输出帧率（从 `02_done/[an-XX-YY]/` 文件夹名解析）
  2. 应用设置中的默认帧率
  3. 硬编码默认值：24fps

#### 背景设置
- **默认**: 黑色背景（`#000000`）
- **可选**: 透明背景（通过应用设置开关控制）
- **设置位置**: 应用设置 → 预览 → "使用透明背景（序列帧预览）"

#### 质量分级
为了性能优化，序列帧预览分为两个质量级别：

| 使用场景 | 质量级别 | 最大宽度 | 用途 | 内存占用（估算） |
|---------|---------|---------|------|-----------------|
| **素材卡片** | 低质量 | 200px | 网格展示大量素材 | ~1.92 MB/预览 |
| **侧边栏** | 高质量 | 800px | 查看详情 | ~10.7 MB/预览 |

**内存管理**:
- 最多缓存10个预览动画（LRU策略）
- 最多加载120帧/预览（超过则降采样）
- 总内存占用预估：~28.8 MB（10个缓存 × 平均2.88MB）

#### 技术实现要点

**Canvas实时渲染**:
```javascript
class SequenceFramePreview {
  constructor(config) {
    this.frameFolder = config.frameFolder;      // 序列帧文件夹路径
    this.fps = config.fps || 24;                // 帧率
    this.quality = config.quality;              // 'low' | 'high'
    this.transparentBg = config.transparentBg;  // 透明背景开关
    this.frames = [];
    this.currentFrame = 0;
    this.playing = false;
  }

  async loadFrames() {
    // 根据质量级别加载帧
    const targetWidth = this.quality === 'low' ? 200 : 800;
    this.frames = await loadFramesScaled(this.frameFolder, targetWidth);
  }

  play() {
    const frameInterval = 1000 / this.fps;
    this.animationTimer = setInterval(() => {
      // 清空画布
      if (this.transparentBg) {
        this.ctx.clearRect(0, 0, this.canvas.width, this.canvas.height);
      } else {
        this.ctx.fillStyle = '#000000';
        this.ctx.fillRect(0, 0, this.canvas.width, this.canvas.height);
      }
      // 绘制当前帧（居中）
      this.ctx.drawImage(this.frames[this.currentFrame], x, y);
      this.currentFrame = (this.currentFrame + 1) % this.frames.length;
    }, frameInterval);
  }
}
```

**性能优化**:
- **懒加载**: 使用Intersection Observer，只加载可见区域的预览
- **暂停机制**: 卡片不可见时暂停动画
- **LRU缓存**: 最多缓存10个预览，超出时清理最久未使用的
- **IndexedDB持久化**: 低质量帧可选持久化缓存

**帧率解析示例**:
```javascript
// 从 02_done/[an-30-12]/ 解析帧率为30fps
function parseFrameRateFromFolder(folderName) {
  const match = folderName.match(/\[an-(\d+)-\d+\]/);
  return match ? parseInt(match[1]) : null;
}
```

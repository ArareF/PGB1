## 🎯 使用场景

### 场景1：新项目创建
- 用户在主页点击 `[新建项目]`
- 输入项目名（如 `218_NewGame`）
- 程序自动创建标准文件夹结构

### 场景2：旧项目导入
- 用户手动将现有项目文件夹（如 `217_RedDevil`）拖入设定目录
- 程序启动或刷新时自动扫描并识别

### 场景3：页面刷新
- 用户在外部修改文件后点击刷新
- 用户切换页面时自动刷新
- 程序失焦后回归时自动刷新

```

03_Render_VFX/VFX/
├─ Export/
│   └─ Prototype/
│       ├─ 00_original/
│       ├─ 01_scale/
│       └─ 02_done/
├─ nextcloud/
│   └─ Prototype/
│       ├─ big_win/
│       │   └─ _original/       ← 自动创建

│       │   └─ _original/

│       │   └─ _original/
│       ├─ spinbutton/
│       │   └─ _original/
│       ├─ symbol/
│       │   └─ _original/
```

- ✅ Prototype任务自动创建7个固定子分类

- ✅ 详细规则参考 `Prototype特例规则.md`

### 项目识别规则

#### 1. 扫描项目根目录

- 扫描用户设置的项目根目录下的所有一级子文件夹
- 忽略隐藏文件夹（`.` 开头）
- 忽略系统文件夹（`.Trash`、`System Volume Information` 等）

#### 2. 判断是否为有效项目

检查文件夹是否包含标志性结构：
```
检查：{文件夹}/03_Render_VFX/VFX/Export/
```

- ✅ **路径存在** → 识别为项目
- ❌ **路径不存在** → 跳过该文件夹

#### 3. 创建或读取配置文件

**如果存在 `.pgb1_project.json`**：
- 直接读取配置

**如果不存在 `.pgb1_project.json`**：
- 自动创建配置文件
- 标记为导入项目：
  ```json
  {
    "project_name": "217_RedDevil",
    "created_at": "2026-02-16T10:30:00Z",
    "imported": true,
    "deadline": null,
    "enabled_tasks": [],  // 待扫描后填充
    "archived_tasks": []
  }
  ```

---

### 任务识别规则

#### 1. 扫描Export目录

扫描 `03_Render_VFX/VFX/Export/` 下的所有文件夹

**忽略规则**：
- 隐藏文件（`.DS_Store`、`Thumbs.db` 等）
- 隐藏文件夹（`.archived_tasks/` 等）
- 系统文件夹

#### 2. 匹配全局任务清单

对每个文件夹进行匹配：

**步骤1：精确匹配（大小写不敏感）**
```
文件夹名: "Free Spin"
全局清单: ["free spin", "ambient", ...]

匹配规则：toLowerCase() 后完全匹配
结果："Free Spin" → 匹配成功 → "free spin"
```

**步骤2：模糊匹配（精确匹配失败时）**
```
文件夹名: "Free Spni" (拼写错误)
全局清单: ["free spin", "ambient", ...]

使用 Levenshtein Distance 算法
相似度阈值: 80%

如果相似度 > 80%:
  ├─ 记录到报告："Free Spni 可能是 Free Spin（相似度 87%）"
  └─ 暂不修改，等待用户确认
```

**步骤3：匹配失败**
```
文件夹名: "Custom Effect"
全局清单: 无匹配

结果:
  ├─ 记录到报告："发现未知任务 Custom Effect"
  └─ 不添加到 enabled_tasks
  └─ 不创建对应的 nextcloud 和 PSD 目录
```

#### 3. 更新配置文件

将匹配成功的任务添加到 `enabled_tasks`：
```json
{
  "enabled_tasks": [
    "free spin",
    "ambient",
    "infoboard",
    "main ui",
    "mouse hover",
    "prototype",
    "spin button",
    "win highlight"
  ]
}
```

旧项目可能缺少某些工作流目录。

**规则**：
- ❌ **不自动创建**缺失的目录

**示例**：

## 🔄 刷新机制

### 刷新触发时机

#### 1. 自动刷新

**页面切换时**：
```
主页 → 项目页
  ↓
重新扫描项目根目录
更新项目列表

项目页 → 任务页
  ↓
重新扫描 Export/{任务名}/
更新素材列表

任务页 → 项目页
  ↓
重新计算任务进度
更新任务卡片

项目页 → 主页
  ↓
重新计算项目进度
更新项目卡片
```

**程序失焦后回归**：
```
用户切到AE导出文件
  ↓
程序失焦

用户切回程序
  ↓
检测到窗口重新获得焦点
  ↓
自动刷新当前页面
```

**程序启动时**：
```
启动程序
  ↓
全局扫描：
  ├─ 扫描项目根目录
  ├─ 识别所有项目
  ├─ 读取所有 .pgb1_project.json
  └─ 构建完整的项目列表

显示主页
```

#### 2. 手动刷新

**刷新入口**：
- 任务页小标题栏：`[🔄刷新]` 按钮
- 辅助页面更多菜单：`[刷新]` 选项

**刷新行为**：
- 重新扫描当前页面对应的目录
- 更新显示内容

### 数据流向

**原则**：数据一层层往上传

**示例场景**：用户上传新素材
```
任务页
  ↓ 用户上传素材
素材状态变化：已缩放 → 已上传
  ↓ 返回项目页
项目页自动刷新
  ↓
重新扫描任务文件夹
重新计算任务进度：1/3 → 2/3
  ↓ 返回主页
主页自动刷新
  ↓
重新扫描所有任务
重新计算项目进度：60% → 80%
```

**全局同步**：
- 程序启动时执行一次全局扫描
- 构建完整的项目结构缓存
- 后续页面切换时基于缓存+增量更新

## 🔍 识别规则细节

### 文件类型识别（静帧 vs 序列帧）

**场景**：扫描 `00_original/` 或 `01_scale/[XX]/` 时识别文件

**识别规则**：
```
1. 提取文件名，按 "_数字.扩展名" 模式分组

示例：
00_original/
├─ effect_01.png       ← 基础名: effect
├─ loop_01.png         ← 基础名: loop
├─ loop_02.png         ← 基础名: loop
├─ loop_04.png         ← 基础名: loop (跳帧)
└─ single.png          ← 基础名: single

2. 按基础名分组：
effect 组: [effect_01.png]
loop 组: [loop_01.png, loop_02.png, loop_04.png]
single 组: [single.png]

3. 判断类型：
- effect 组: 只有1个文件且以_01结尾 → 静帧（规范化后去掉_01）
- loop 组: 有多个文件(_01, _02, _04) → 序列帧
- single 组: 只有1个文件且不以_01结尾 → 静帧
```

**跳帧处理**：
- 宽松接受：只要有 `_01`, `_02` ... 的模式，识别为序列帧
- 跳帧不影响识别
- 在侧边栏显示警告："帧数: 3帧（缺少第3帧）"

---

### 缩放比例提取

**从文件夹名提取缩放比例**

**标准格式**：
- `[img-100]` → 静帧，100%
- `[an-70-24]` → 序列帧，70%，24fps

**非标准格式模糊匹配**：
```
[100]           → 缺少类型前缀，提取 100%，记录警告
img-50          → 缺少方括号，提取 50%，记录警告
[an-30]         → 序列帧缺少帧率，提取 30%，帧率标记"未知"
[img-35]        → 非标准档位，提取 35%，标记"自定义比例"
```

**处理策略**：
- 自动提取比例值
- 记录到扫描报告
- 不自动修正，等待用户查看报告后决定

---

### 素材进度状态识别

**判定优先级**（从高到低）：
```
1. nextcloud/{任务名}/文件名.ext 存在 → "已上传"
2. 02_done/[img-XX]/文件名.webp 存在 → "已完成"
3. 01_scale/[XX]/文件名.ext 存在 → "已缩放"
4. 00_original/文件名.ext 存在 → "原始文件"
5. 都不存在 → "未开始"（不应该出现）
```

**详细规则** → 参考 `文件命名与组织规则.md` - 素材进度追踪系统

## ⚠️ 异常处理

### 权限错误

**场景**：扫描时无法读取某个文件夹

**处理**：
```
捕获异常
  ↓
标记该文件夹为 "无法访问"
  ↓
记录到日志
  ↓
继续扫描其他文件夹
  ↓
扫描完成后在报告中显示：
"⚠️ 部分文件夹无法访问，请检查权限"
```

---

### 长耗时反馈

**场景**：扫描大型项目耗时较长

**处理**：
```
开始扫描
  ↓
500ms后仍未完成
  ↓
显示加载动画：
┌────────────────────────────────────────┐
│          正在加载项目...               │
│                                        │
│            ⏳ 加载中                   │
└────────────────────────────────────────┘

如果超过10秒
  ↓
显示进度和取消按钮：
┌────────────────────────────────────────┐
│          正在扫描项目...               │
│                                        │
│     已扫描 3/8 个任务                  │
│     ⏳ 加载中...                       │
│                                        │
│              [取消]                    │
└────────────────────────────────────────┘

用户点击取消
  ↓
停止扫描
显示已扫描的部分
```

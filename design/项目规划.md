## 📋 项目概述

### 核心定位
- **产品名称**: PGB1（文件整理工具）
- **核心功能**: 快捷方便的整理文件
- **目标用户**: 2D游戏特效师
- **使用场景**: 将制作完成的素材快速整理后上传到网盘

### 典型工作流
```
AE/TexturePacker等工具
  ↓ 导出文件到本地
PGB1程序
  ↓ 快速整理分类
浏览器网盘
  ↓ 拖拽上传
完成
```

### 原型版本的问题
- ❌ 技术栈：Electron不支持文件拖拽到外部程序（致命缺陷）
- ❌ 界面设计不合理（具体待补充）
- ❌ 模块化不足，维护困难

## 🔧 技术栈

### ✅ 确定方案：Tauri
- **前端**: Vue 3 + HTML/CSS/JavaScript
- **后端**: Rust
- **目标平台**: Windows（优先），未来扩展Mac

### 核心优势
- ✅ 支持文件拖拽到外部浏览器（Electron做不到）
- ✅ 毛玻璃效果用CSS实现，灵活易调
- ✅ 无边框窗口、悬浮窗、系统托盘原生支持
- ✅ 内嵌网页零成本（本身就是WebView）
- ✅ 安装包小（3-5MB）、性能好、跨平台

## 📂 文件整理逻辑

### 分类层级结构（实际为四层工作流）

基于实际项目 217_RedDevil 的结构：

```
项目（一层）
  └─ 功能分类（二层）
      └─ 工作流阶段（三层）
          └─ 文件类型 + 缩放比例（四层）
```

**第一层：项目**
- 格式：`{项目编号}_{项目代号}`
- 示例：`217_RedDevil`
- 包含子目录：
  - `00_Game Design & Doc` - 游戏设计和文档
  - `01_Preproduction` - 预制作（设计稿）
  - `02_Production` - 制作中间文件
  - `03_Render_VFX` - **渲染和特效（主要工作目录）**
  - `04_Trailer` - 宣传片素材
  - `05_Outside` - 外部资源

**第二层：功能分类**
- 位置：`03_Render_VFX/VFX/Export/`
- 已识别的分类：
  - `Ambient` - 环境特效
  - `Free Spin` - 免费旋转特效
  - `Infoboard` - 信息板
  - `Main UI` - 主界面UI
  - `Mouse Hover` - 鼠标悬停效果
  - `Prototype` - 原型素材
  - `Spin Button` - 旋转按钮
  - `Win Highlight` - 赢得高亮效果

**第三层：工作流阶段**（每个功能分类下的标准子目录）
- `00_original/` - 原始文件（AE/TexturePacker导出）
- `01_scale/` - 按缩放比例分类的文件
- `02_done/` - 最终完成的文件
- `03_preview/` - 预览视频

**第四层：文件类型 + 缩放比例**
在 `01_scale/` 和 `02_done/` 中按此分类：
- **缩放比例目录**（01_scale）:
  - `[100]` - 100%原始大小
  - `[70]` - 70%
  - `[50]` - 50%
  - `[40]` - 40%

- **类型+缩放目录**（02_done）:
  - `[img-100]` - 静帧，100%
  - `[img-70]` - 静帧，70%
  - `[img-50]` - 静帧，50%
  - `[an-100-24]` - 序列帧，100%，24帧
  - `[an-50-18]` - 序列帧，50%，18帧
  - `[an-40-18]` - 序列帧，40%，18帧

**文件类型说明**：
- **静帧（img）**: 单张图片
- **序列帧（an）**: 连续的多帧图片，会被TexturePacker打包成精灵图

**缩放比例说明**：
- **重要**: 不是文件体积，而是素材的缩放处理标记
- **目的**: 防止忘记对素材做了多少缩放
- **档位**: 100%、70%、50%、40%（根据素材重要性选择）

### 📌 双视图模式（两个都要）

#### 模式1：树形分类视图
```
项目A
  ├─ 功能分类1
  │   ├─ 静帧
  │   │   ├─ 100%
  │   │   ├─ 70%
  │   │   └─ 50%
  │   └─ 序列帧
  │       ├─ 100%
  │       └─ 70%
  └─ 功能分类2
      └─ ...
```
- **特点**: 层级清晰，便于按分类浏览
- **用途**: 整理、分类、批量操作

#### 模式2：名称排序视图（带标签）
```
[卡片1] 素材名称A    [静帧] [70%]
[卡片2] 素材名称B    [序列帧] [100%]
[卡片3] 素材名称C    [静帧] [50%]
...（按名称字母/拼音顺序排列）
```
- **特点**: 扁平化展示，按名称排序
- **元数据**: 文件类型和缩放比例用标签显示在卡片上
- **用途**: 快速查找、预览、拖拽上传

**两种模式关系**:
- 同一套数据，两种展示方式
- 可快速切换（Tab或按钮切换）
- 待明确：两种模式下的操作是否一致？

### 文件来源与类型
- **AE导出**: 视频、图像序列
- **TexturePacker导出**: 精灵图（spritesheet）+ 配置文件（JSON/XML）+ 工程文件（.tps）
- **文档**: DOC/Word（游戏说明文档）

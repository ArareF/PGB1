# PGB1 打包发布指南

> 给未来 Agent：本文档记录完整的 Windows 安装包构建流程、配置项说明和已知坑点。
> 执行打包前请先确认版本号已正确更新（见第 3 节）。

---

## 1. 完整发版流程（速查）

每次发新版按此顺序执行：

```
1. 更新版本号（4 处同步，见第 3 节）
2. git commit + push
3. PowerShell 打包（见第 2 节）
4. 创建 GitHub Release + 上传文件（见第 9 节）
```

---

## 2. 打包命令

**必须在 PowerShell 中执行**（bash 环境变量传递有问题）：

```powershell
cd "C:\work\PG Butler\PGB1"
$env:TAURI_SIGNING_PRIVATE_KEY = (Get-Content "$env:USERPROFILE\.tauri\PGB1.key" -Raw).Trim()
$env:TAURI_SIGNING_PRIVATE_KEY_PASSWORD = "<你的私钥密码>"
npm run tauri build
```

输出位置：
```
src-tauri/target/release/bundle/nsis/PG素材管理系统_<version>_x64-setup.exe
src-tauri/target/release/bundle/nsis/PG素材管理系统_<version>_x64-setup.exe.sig
```

前提条件：
- Node.js + npm 已安装
- Rust 工具链已安装（`rustup`）
- 签名私钥存在于 `C:\Users\mindb\.tauri\PGB1.key`
- 首次打包需要等待 Cargo 下载依赖（约 10 分钟），后续增量编译约 1–2 分钟

**踩坑记录：**
- ❌ bash 中 `export TAURI_SIGNING_PRIVATE_KEY=...` 传不进 npm 子进程，签名静默跳过或密码错误
- ❌ `Get-Content` 不加 `.Trim()` 会带尾部换行，导致密钥解析失败
- ❌ 空密码生成的密钥在签名阶段无法验证（原因未明），建议始终设密码

## 3. 版本号更新（发布新版时必做）

需要同步修改 **四处**，保持一致：

| 文件 | 字段 | 示例 |
|------|------|------|
| `package.json` | `"version"` | `"2.6.0"` |
| `src-tauri/tauri.conf.json` | `"version"` | `"2.6.0"` |
| `src-tauri/Cargo.toml` | `version` | `"2.6.0"` |
| `src/config/app.ts` | `APP_VERSION` | `'V2.6.0'` |

版本号命名约定：`2.x.x`（大版本固定 2，功能迭代加中间位，bugfix 加末位）

---

## 4. 打包目标配置

`src-tauri/tauri.conf.json` 的 bundle 节：

```json
"bundle": {
  "targets": "nsis",
  "createUpdaterArtifacts": true,
  ...
}
```

- `"targets": "nsis"` — 必须是 nsis，不能是 all 或 msi（WiX 不支持中文 productName）
- `"createUpdaterArtifacts": true` — 启用自动更新签名，打包时生成 `.exe.sig`

---

## 5. 自动更新配置

`src-tauri/tauri.conf.json` 的 plugins 节：

```json
"plugins": {
  "updater": {
    "endpoints": [
      "https://github.com/ArareF/PGB1/releases/latest/download/latest.json"
    ],
    "pubkey": "<公钥内容>"
  }
}
```

- **公钥**：`C:\Users\mindb\.tauri\PGB1.key.pub` 的内容
- **私钥**：`C:\Users\mindb\.tauri\PGB1.key`（⚠️ 绝对不能提交到仓库或泄露，丢失后无法签名新版本）
- **私钥密码**：本地保管，不写入仓库
- **GitHub 仓库**：`https://github.com/ArareF/PGB1`（私有）

### 签名密钥重生成（仅在密钥丢失时）

```powershell
cd "C:\work\PG Butler\PGB1"
npx tauri signer generate -w "$env:USERPROFILE\.tauri\PGB1.key" --force
```

重生成后必须：
1. 将新公钥更新到 `tauri.conf.json` 的 `plugins.updater.pubkey`
2. 旧版本用户将无法验证新版签名（updater 会静默失败，用户需手动下载安装包）

## 6. 图标管理

应用图标源文件：`icon.ico`（项目根目录）

生成所有平台所需尺寸：
```bash
npx tauri icon icon.ico
```

输出到 `src-tauri/icons/`，已包含 Windows ICO、macOS ICNS、各分辨率 PNG。

**图标修改流程：** 替换 `icon.ico` → 重新运行上述命令 → 重新打包。

## 8. CSP 配置说明

`src-tauri/tauri.conf.json` → `app.security.csp`，当前值：

```
default-src 'self' ipc: http://ipc.localhost https:;
img-src 'self' asset: http://asset.localhost blob: data: https:;
media-src 'self' asset: http://asset.localhost blob: data:;
script-src 'self' 'unsafe-inline' https:;
style-src 'self' 'unsafe-inline' https:;
connect-src 'self' https:;
frame-src 'self' asset: http://asset.localhost
```

**关键规则（踩过的坑）：**
- `media-src` 必须单独声明并包含 `asset:`：`<video>` 不继承 `img-src`，缺失打包后视频全灭
- `img-src` 必须包含 `data:`（PSD 缩略图）和 `blob:`（Blob URL 图片）
- **不要**把 `targets` 改成 `all`，会触发 WiX 中文 productName bug

## 9. GitHub Release 发布

### 前置：安装并登录 gh CLI

```powershell
# 已安装：C:\Program Files\GitHub CLI\gh.exe
# 登录（仅首次）：
& "C:\Program Files\GitHub CLI\gh.exe" auth login
```

### 创建 Release

打包完成后，执行以下命令创建 Release 并上传文件：

**第 1 步：准备 latest.json**

```json
{
  "version": "<新版本号>",
  "notes": "更新说明",
  "pub_date": "<ISO 8601 时间>",
  "platforms": {
    "windows-x86_64": {
      "url": "https://github.com/ArareF/PGB1/releases/download/v<版本>/PG素材管理系统_<版本>_x64-setup.exe",
      "signature": "<.exe.sig 文件的完整内容>"
    }
  }
}
```

- `signature` 字段：把 `.exe.sig` 文件的**全部内容**粘贴进去（一行 base64 字符串）
- `url`：指向同一 Release 中的 .exe 文件

**第 2 步：用 gh 创建 Release**

```bash
gh release create v<版本号> \
  "src-tauri/target/release/bundle/nsis/PG素材管理系统_<版本>_x64-setup.exe" \
  "latest.json" \
  --title "v<版本号> — 更新标题" \
  --notes "更新说明"
```

发布后验证：`gh release view v<版本号>`

## 10. 覆盖安装

NSIS 安装包支持覆盖安装：直接运行新版 `.exe` 即可更新，不需要先卸载。用户数据（`%APPDATA%\com.pgsoft.pgb1\`）不受影响。

自动更新（v2.6.0+）：用户启动旧版后 3 秒自动检查 GitHub Release，发现新版弹窗提示。用户可选择「立即更新」「稍后提醒」「跳过此版本」。

# PGB1 打包发布指南

> 给未来 Agent：本文档记录完整的 Windows 安装包构建流程、配置项说明和已知坑点。
> 执行打包前请先确认版本号已正确更新（见第 3 节）。

---

## 1. 打包命令

```bash
npm run tauri build
```

输出位置：
```
src-tauri/target/release/bundle/nsis/PG素材管理系统_<version>_x64-setup.exe
```

前提条件：
- Node.js + npm 已安装
- Rust 工具链已安装（`rustup`）
- 首次打包需要等待 Cargo 下载依赖（约 10 分钟），后续增量编译约 1 分钟

---

## 2. 打包目标配置

`src-tauri/tauri.conf.json` 的 bundle 节：

```json
"bundle": {
  "targets": "nsis",          ← 必须是 nsis，不能是 all 或 msi
  "icon": [...],
  "windows": {
    "nsis": {
      "languages": ["SimpChinese"],          ← 中文界面
      "installMode": "currentUser",          ← 免 UAC，安装到用户目录
      "headerImage": "installer_header.bmp", ← 安装步骤页顶部横幅（150×57 px BMP）
      "sidebarImage": "installer_sidebar.bmp", ← 欢迎/完成页侧边图（164×314 px BMP）
      "installerIcon": "icons/icon.ico"      ← 安装包 .exe 的文件图标
    }
  }
}
```

**为什么只用 NSIS，不用 WiX：** WiX 3（Tauri 捆绑的旧版）无法处理非 ASCII 的 productName，会直接报错。中文产品名只能走 NSIS。

## 3. 版本号更新（发布新版时必做）

需要同步修改三处，保持一致：

| 文件 | 字段 | 示例 |
|------|------|------|
| `package.json` | `"version"` | `"2.0.1"` |
| `src-tauri/tauri.conf.json` | `"version"` | `"2.0.1"` |
| `src-tauri/Cargo.toml` | `version` | `"2.0.1"` |

版本号命名约定：`2.x.x`（大版本固定 2，功能迭代加中间位，bugfix 加末位）

---

## 4. 图标管理

应用图标源文件：`icon.ico`（项目根目录）

生成所有平台所需尺寸：
```bash
npx tauri icon icon.ico
```

输出到 `src-tauri/icons/`，已包含 Windows ICO、macOS ICNS、各分辨率 PNG。

**图标修改流程：** 替换 `icon.ico` → 重新运行上述命令 → 重新打包。

---

## 5. 安装程序自定义图片

| 图片 | 文件 | 尺寸 | 出现位置 |
|------|------|------|---------|
| 顶部横幅 | `src-tauri/installer_header.bmp` | 150×57 px | 各安装步骤页右上角 |
| 侧边栏 | `src-tauri/installer_sidebar.bmp` | 164×314 px | 欢迎页和完成页左侧 |

**格式要求：BMP**（NSIS 不支持 PNG，会 warning 5040 并忽略图片）。

更新流程：直接替换对应 BMP 文件，重新打包即可。

如果只有 PNG，用 PowerShell 转换：
```powershell
Add-Type -AssemblyName System.Drawing
$img = [System.Drawing.Image]::FromFile("source.png")
$bmp = New-Object System.Drawing.Bitmap($img)
$bmp.Save("output.bmp", [System.Drawing.Imaging.ImageFormat]::Bmp)
$img.Dispose(); $bmp.Dispose()
```

## 6. CSP 配置说明

`src-tauri/tauri.conf.json` → `app.security.csp`，当前值：

```
default-src 'self' ipc: http://ipc.localhost https:;
img-src 'self' asset: http://asset.localhost blob: data: https:;
media-src 'self' asset: http://asset.localhost blob: data:;
script-src 'self' 'unsafe-inline' https:;
style-src 'self' 'unsafe-inline' https:;
connect-src 'self' https:;
frame-src 'self' asset: http://asset.localhost
```

**关键规则（踩过的坑）：**
- `img-src` 必须包含 `asset: http://asset.localhost`：asset 协议图片（convertFileSrc 结果）
- `img-src` 必须包含 `data:`：PSD 缩略图、canvas.toDataURL() 结果
- `img-src` 必须包含 `blob:`：Blob URL 图片
- `media-src` 必须单独声明并包含 `asset:`：`<video>` 和 `<audio>` 不继承 `img-src`，缺失会导致视频完全无法加载（开发模式不报错，打包后才暴露）
- **不要**把 `targets` 改成 `all`，会触发 WiX 中文 productName bug

## 7. 覆盖安装 / 静默更新

NSIS 安装包支持覆盖安装：直接运行新版 `.exe` 即可更新，不需要先卸载。用户数据（`%APPDATA%\com.pgsoft.pgb1\`）不受影响。

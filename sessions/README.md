# 会话记忆目录 (Sessions)

本目录存放**项目会话记录和历史日志**，用于追踪需求讨论过程和项目进展。

## 📂 目录内容

### `session_YYYY-MM-DD.md` - 会话日志
- **用途**：记录每次需求讨论会话的详细内容
- **格式**：按日期命名（如 `session_2026-02-13.md`）
- **包含内容**：
  - 已确定的决策
  - 待讨论的重点内容
  - 讨论中的问题
  - 下一步计划

### `project_history.md` - 项目历史日志
- **用途**：记录项目从启动到完成的所有关键节点和里程碑
- **格式**：时间线格式
- **包含内容**：
  - 项目阶段划分
  - 关键里程碑
  - 重要决策点
  - 整体进度追踪

---

## 🔄 与设计文档的关系

- **本目录（sessions/）**：记录"我们讨论了什么"、"何时做出的决策"
- **设计目录（design/）**：记录"最终确定的需求和规范"

设计文档是会话讨论的结晶，会话日志是设计文档的演化历史。

---

## 📝 使用规范

### 对于产品总监
- 可以随时查看历史会话，回顾讨论细节
- 可以追溯某个决策是在哪次会话中确定的

### 对于未来的Agent
- **首先阅读**：`../design/` 中的设计文档（最新需求）
- **必要时参考**：本目录的会话日志（了解决策背景和演化过程）
- **不要修改**：历史会话日志应保持不变（只追加，不修改）

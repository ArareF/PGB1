# PGB1 开发规范

本文档是编程阶段必须遵守的代码规范和质量标准。
**适用范围**: 编程阶段（已进入）。

---

## 🚫 绝对禁止

### 硬编码与魔法数值
- ❌ 严禁硬编码颜色、尺寸、间距
- ❌ 必须建立设计系统（DesignSystem.css 或 ThemeConfig.js）
- ❌ 所有配置（功能分类、缩放比例档位、文件类型等）必须提取到独立配置文件

**正确示例**:
```css
/* DesignSystem.css */
:root {
  --glass-bg: rgba(255, 255, 255, 0.1);
  --glass-blur: 10px;
  --primary-color: #4A90E2;
}

.glass-panel {
  background: var(--glass-bg);
  backdrop-filter: blur(var(--glass-blur));
}
```

**错误示例**:
```css
/* ❌ 到处硬编码 */
.panel { background: rgba(255, 255, 255, 0.1); }
.header { background: rgba(255, 255, 255, 0.15); }
```

---

### 破坏性修改
- ❌ 未经允许不得修改未被提及的文件
- ❌ 严禁顺手格式化无关代码（会污染Git Diff）
- ❌ 修改通用组件前必须预演爆炸半径

---

### 静默失败
- ❌ try-catch后必须打日志或抛异常
- ❌ 不得假装无事发生

**正确示例**:
```rust
match result {
    Ok(data) => data,
    Err(e) => {
        eprintln!("文件读取失败: {}", e);
        return Err(e);
    }
}
```

## 📁 代码结构

### 目录规划
```
src/
├── components/     # UI组件（独立、可复用）
├── views/          # 视图（树形分类视图、名称排序视图）
├── services/       # 业务逻辑（文件整理、分类、拖拽）
├── models/         # 数据模型
├── config/         # 配置文件
├── utils/          # 工具函数
└── styles/         # 全局样式（含DesignSystem）
```

**原则**：
- 模块边界清晰
- 避免循环依赖
- 单向数据流

## 🔤 代码规范

### 命名规则
- **Rust**: snake_case（函数、变量）、PascalCase（结构体、枚举）
- **JavaScript**: camelCase（变量、函数）、PascalCase（组件）
- **CSS**: kebab-case（类名）
- **必须语义化**，拒绝a、b、tmp等无意义命名

---

### 注释规范
- **面向产品总监的解释必须使用简体中文**
- 复杂逻辑必须注释"为什么"，而非"是什么"
- 配置文件必须有注释说明

**示例**:
```rust
// ❌ 错误：描述"是什么"
// 循环遍历文件
for file in files { ... }

// ✅ 正确：解释"为什么"
// 需要过滤掉隐藏文件，因为特效师通常不需要整理.DS_Store等系统文件
for file in files.iter().filter(|f| !f.is_hidden()) { ... }
```

---

### 错误处理

**Rust后端**:
```rust
fn read_file(path: &str) -> Result<String, String> {
    std::fs::read_to_string(path)
        .map_err(|e| format!("读取文件失败: {}: {}", path, e))
}
```

**JavaScript前端**:
```javascript
async function loadFiles() {
  try {
    const files = await invoke('read_files');
    return files;
  } catch (error) {
    console.error('加载文件失败:', error);
    showErrorToast('无法加载文件，请检查路径是否正确');
    return [];
  }
}
```

## 🎨 UI 开发理念（已验证）

以下原则从 Phase 1 主界面搭建中总结，所有 agent 必须遵循。

### 样式零硬编码

**所有**视觉属性必须引用 `design-system.css` 中的 token，包括：
- 颜色 → `var(--text-primary)`, `var(--glass-medium-bg)` 等
- 间距 → `var(--spacing-*)` 系列
- 圆角 → `var(--radius-*)` / `var(--floating-navbar-radius)` 等语义别名
- 字号 → `var(--text-*)` 系列
- 字重 → `var(--font-weight-*)` 系列
- 阴影 → `var(--shadow-*)` 系列
- 动画 → `var(--transition-*)`, `var(--duration-*)` 系列

**需要新数值时的做法**：先在 `design-system.css` 的 `:root` 中定义 token，再在组件中引用。同时更新 `design/DesignSystem.md` 文档。

```css
/* ✅ 正确：新增 token 后引用 */
:root { --window-close-bg: rgba(183, 28, 28, 0.8); }
.close { background: var(--window-close-bg); }

/* ❌ 错误：直接写值 */
.close { background: rgba(183, 28, 28, 0.8); }
```

### 视觉对齐原则

- **可见边界对齐**：对齐的是毛玻璃容器的可见边缘，不是外层透明容器的边缘。如果容器有"出血"padding，对齐时需要补偿
- **间距一致性**：同层级区域之间的间距（水平和垂直）应保持一致。例如 top-row 和 content-row 的垂直 gap = content-row 内部的水平 gap
- **圆角统一**：同层级的悬浮岛/容器使用同一个圆角 token（`--floating-navbar-radius`），不允许混用

### 悬浮岛设计语言

整个界面基于"悬浮岛"概念 — 独立的毛玻璃容器漂浮在 Acrylic 背景上。

**核心规则**：
- 所有悬浮岛使用 `glass-medium` 类获取毛玻璃效果
- 圆角统一用 `var(--floating-navbar-radius)` = `var(--radius-floating-island)` = 20px
- 悬浮岛之间留有呼吸间距，不紧贴
- 新增独立按钮类悬浮岛时，使用 `--floating-action-*` token 系列

### Token 层级结构

```
原子值（不直接用于组件）
  --radius-2xl: 20px
  --spacing-5: 20px
    ↓
语义别名（优先使用）
  --radius-floating-island: var(--radius-2xl)
  --floating-navbar-radius: var(--radius-floating-island)
    ↓
组件级 token（特定组件专用）
  --floating-action-radius: var(--floating-navbar-radius)
  --window-control-radius: var(--radius-full)
```

**使用原则**：组件样式中优先用语义别名或组件级 token，不直接引用原子值。

### 组件自洽原则

每个 Vue 组件的 `<style scoped>` 应该：
- 只负责自身的内部布局和视觉
- 通过 `width: 100%` / `height: 100%` / `flex` 等让父级控制外部尺寸
- 不给自己加 margin（由父级的 gap/padding 控制间距）
- 不硬编码高度，除非有明确的 token 指定（如 `--floating-action-height`）

### 设计→代码→文档同步

任何视觉调整都必须三方同步：
1. `design/DesignSystem.md` — 设计规范文档
2. `src/styles/design-system.css` — CSS token 定义
3. 组件文件 — 引用 token

新增 token 时，先写文档，再写 CSS，最后在组件中使用。

### 当前 Token 速查

| 类别 | Token | 值 | 用途 |
|------|-------|-----|------|
| 悬浮岛圆角 | `--floating-navbar-radius` | 20px | 所有悬浮岛统一圆角 |
| 悬浮操作按钮 | `--floating-action-height` | 48px | 更多菜单等独立按钮高度 |
| 悬浮操作按钮 | `--floating-action-radius` | 20px | 同悬浮岛圆角 |
| 悬浮操作按钮 | `--floating-action-padding-x` | 20px | 水平内距 |
| 窗口控制 | `--window-control-size` | 32px | 控制按钮尺寸 |
| 窗口控制 | `--window-control-radius` | 9999px | 圆形 |
| 窗口控制 | `--window-control-bg` | glass-medium-bg | 常显背景 |
| 窗口关闭 | `--window-close-bg` | 暗红 | 关闭按钮常态色 |
| 快捷方式栏 | `--floating-sidebar-width` | 100px | 侧边栏宽度 |
| 快捷方式栏 | `--floating-sidebar-item-size` | 60px | 图标按钮尺寸 |
